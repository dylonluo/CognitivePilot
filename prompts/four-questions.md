# 四问框架提示词

用于快速决策和技术问题分析（5-8分钟）

## 分析步骤

### 1. 本质是什么？（第一性原理）
- 剥离所有假设和表象
- 回到问题的根本
- 问：如果从零开始，这个问题是什么？

### 2. 关键在哪？（二八法则）
- 找到产生80%结果的20%因素
- 识别关键少数
- 问：如果只能解决一个点，应该是什么？

### 3. 根因是什么？（5Why）
- 连续追问5次"为什么"
- 找到真正的根本原因
- 不要停留在表面症状

### 4. 优先级如何？（艾森豪威尔矩阵）
- 重要且紧急 → 立即做
- 重要不紧急 → 计划做
- 紧急不重要 → 委托做
- 不重要不紧急 → 不做

## 输出格式

```markdown
## 场景判断
问题类型：技术/执行
复杂度：简单
影响范围：低
选用框架：四问

## 四问分析

### 1. 本质
[问题的核心是什么]

### 2. 关键
[20%的关键因素]

### 3. 根因
为什么1 → 为什么2 → 为什么3 → 为什么4 → 根因

### 4. 优先级
**立即做**：[...]
**计划做**：[...]

## 决策点
[需要用户决定的事项]
```

## 适用场景

- 技术 bug 排查
- 简单的工作决策
- 日常问题解决
- 快速优先级判断
