# 六维框架提示词

标准分析框架，适用于大多数问题（15分钟）

## 分析步骤

### 1. 定义问题（第一性原理）
- 这个问题的本质是什么？
- 剥离所有假设
- 如果从零开始会怎么看？

### 2. 找主要矛盾（矛盾分析法）
- 主要矛盾是什么？
- 矛盾的两个方面分别是什么？
- 哪个是矛盾的主要方面？

### 3. 结构化拆解（MECE）
- 将问题分解为相互独立、完全穷尽的子问题
- 确保没有遗漏，没有重叠
- 建立清晰的层次结构

### 4. 多角度思考（六顶思考帽）
- 白帽：客观事实和数据
- 红帽：情感和直觉
- 黑帽：风险和问题
- 黄帽：优势和机会
- 绿帽：创新和可能性
- 蓝帽：流程和控制

### 5. 辩证综合（正反合）
- 正题：当前的观点/方案
- 反题：对立的观点/问题
- 合题：更高层次的综合方案

### 6. 优先排序（艾森豪威尔矩阵）
- 按重要性和紧急性排序
- 明确行动优先级

## 输出格式

```markdown
## 场景判断
问题类型：[人际/业务/...]
复杂度：复杂
影响范围：中
选用框架：六维

## 问题本质（第一性原理）
[核心问题描述]

## 关键矛盾（矛盾分析法）
- 主要矛盾：[...]
- 矛盾的主要方面：[...]
- 次要矛盾：[...]

## 结构化拆解（MECE）
1. [子问题1]
   - 1.1 [...]
   - 1.2 [...]
2. [子问题2]
3. [子问题3]

## 多维视角（六顶思考帽）
- 白帽：[客观数据和信息]
- 红帽：[情绪和直觉反应]
- 黑帽：[可能的问题和风险]
- 黄帽：[优势和积极面]
- 绿帽：[新的可能性和创意]
- 蓝帽：[如何推进和控制]

## 辩证综合（正反合）
- 正题：[当前方案/观点]
- 反题：[对立方案/问题]
- 合题：[综合后的更优方案]

## 行动建议（优先级矩阵）
### 立即做（重要且紧急）
- [...]

### 计划做（重要不紧急）
- [...]

### 委托做（紧急不重要）
- [...]

## 决策点
[需要用户做出的关键决策]
```

## 适用场景

- 人际关系问题
- 业务决策
- 中等复杂度的工作问题
- 需要多角度思考的场景
- 默认框架（不确定时使用六维）
